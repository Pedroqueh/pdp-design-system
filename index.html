Botones Estable

PDP Suite usa tres tipos de botón con una altura recomendada de 32px: principal (fondo #0070F2, texto #F8FAFC), secundario (fondo #F8FAFC, texto #0070F2) e ícono (sin fondo, ícono #0070F2). En conjuntos de acción el orden es secundario → principal, con el principal a la derecha.

Botón principal — una acción primaria por contexto

Fondo #0070F2 y texto #F8FAFC. Representa la acción más importante de la pantalla o sección (Guardar, Finalizar, Crear). Debe haber solo uno por grupo de acciones.

Botón secundario — acciones complementarias

Fondo #F8FAFC, texto #0070F2 y borde sutil. Para acciones de apoyo (Cancelar, Volver, Guardar progreso). Pueden coexistir varios en un mismo grupo.

Botón ícono — acciones compactas

Sin fondo, ícono #0070F2. Para acciones por fila en tablas o barras con poco espacio (Ver, Editar, Eliminar). Siempre debe llevar tooltip para describir su función.

Orden y altura

Altura recomendada de 32px para los tres tipos. En conjuntos de acción el orden es secundario(s) → principal, alineados a la derecha y con el principal en el extremo.

Estado deshabilitado

Cuando una acción no está disponible, el botón mantiene su forma con opacidad reducida y cursor not-allowed, sin efecto hover.

XML View — tipos de botón

<!-- Principal: fondo #0070F2, texto #F8FAFC -->

<!-- Secundario: fondo #F8FAFC, texto #0070F2 -->
<Button text="Cancelar" type="Default" press=".onCancel" />

<!-- Ícono: sin fondo, ícono #0070F2 (siempre con tooltip) -->
<Button icon="sap-icon://show" type="Transparent"
  tooltip="Ver" press=".onView" />

Filtros Estable

Barra de filtros que permite al usuario acotar los registros visibles en tablas y listados. Los filtros se aplican automáticamente al perder el foco (onBlur), sin necesidad de un botón de búsqueda explícito. El enlace Limpiar filtros se activa solo cuando al menos un filtro tiene valor.

Filtrado automático al perder el foco (onBlur)

Cuando el usuario termina de escribir en el campo de texto o selecciona una opción en el dropdown y pasa a otro elemento, el filtro se ejecuta automáticamente. No se requiere presionar Enter ni ningún botón de búsqueda.

Activación del enlace "Limpiar filtros"

El enlace Limpiar filtros aparece en estado inactivo (azul con 40% de opacidad, no clickeable) cuando ningún filtro tiene valor. Se activa (azul sólido, clickeable) en el momento en que cualquier filtro recibe un valor, sin esperar el onBlur.

Limpiar filtros — restablecer estado

Al hacer clic en Limpiar filtros, todos los campos vuelven a su estado vacío, la tabla muestra todos los registros y el enlace regresa a su estado inactivo.

Filtros combinados (AND)

Cuando el usuario activa más de un filtro, la lógica es AND: solo se muestran registros que cumplan con todos los criterios simultáneamente.

Controller — lógica de filtrado

// Ejecutar al perder foco (onBlur) o al cambiar Select
onSearch: function () {
  var sNombre = this.byId("inputNombre").getValue().toLowerCase();
  var sPais  = this.byId("selectPais").getSelectedKey();

  var oFilters = [];

  if (sNombre) {
    oFilters.push(new Filter("nombre",
      FilterOperator.Contains, sNombre));
  }

  if (sPais) {
    oFilters.push(new Filter("pais",
      FilterOperator.EQ, sPais));
  }

  // AND combinado
  var oCombined = oFilters.length
    ? new Filter({ filters: oFilters, and: true })
    : null;

  this.byId("tabla").getBinding("items").filter(oCombined);

  // Activar/desactivar "Limpiar filtros"
  var bActive = !!(sNombre || sPais);
  this.byId("linkLimpiar").setEnabled(bActive);
},

onClearFilters: function () {
  this.byId("inputNombre").setValue("");
  this.byId("selectPais").setSelectedKey("");
  this.onSearch();
}

Tablas Estable

Patrón estándar para listados y registros en PDP Suite. La tabla vive dentro de una tarjeta con encabezado (título, subtítulo y acción primaria), filas alternadas para mejorar la lectura, y una columna de acciones por fila (Ver, Editar, Eliminar). Los valores vacíos se muestran como -.

Encabezado con acción primaria

Cada tabla vive en una tarjeta con título, subtítulo descriptivo y un botón primario de creación alineado a la derecha (por ejemplo, Crear formulario).

Filas alternadas (zebra) y hover

Las filas pares llevan un fondo gris suave (--surface) para facilitar la lectura horizontal. Al pasar el cursor, la fila se resalta en azul claro (--blue-light).

Columna de acciones por fila

Alineada a la derecha, ofrece Ver (sap-icon://show), Editar (sap-icon://edit) y Eliminar (sap-icon://delete). La acción destructiva resalta en rojo al pasar el cursor.

Valores vacíos y descripciones largas

Cuando un campo no tiene valor se muestra un guion - en color atenuado. Las descripciones extensas se truncan con elipsis para mantener la altura de fila constante.

Tabla con scroll horizontal

En viewports angostos la tabla habilita scroll horizontal dentro de la tarjeta, manteniendo el encabezado de la tarjeta siempre visible.

Formularios Estable

Patrón para la captura de datos en PDP Suite. Todo formulario tiene un título y subtítulo alineados a la izquierda que envuelven los campos. Los labels van sobre el input, los campos obligatorios llevan un asterisco rojo (*) y todo input tiene un placeholder descriptivo. Se presenta en vista completa (acciones en la cabecera) o en modal (acciones en el footer); en ambos el orden es secundario → principal, alineado a la derecha.

Título y subtítulo que envuelven los campos

Todo formulario abre con un título y un subtítulo alineados a la izquierda. El subtítulo usa --slate (no azul, que se reserva para lo navegable). Estos textos enmarcan el conjunto de campos de la sección.

Label arriba, obligatorios con asterisco rojo

El label siempre va sobre el input. Los campos obligatorios muestran un asterisco * (--required) inmediatamente después del texto del label.

Placeholder descriptivo y tooltip opcional

Cada input lleva un placeholder en itálica que orienta sobre lo que se espera. Opcionalmente, un ícono de información junto al label despliega un tooltip con ayuda complementaria (ver campos Responsable y RUT).

Ubicación de las acciones — vista completa vs. modal

En vista completa las acciones van en la cabecera de la sección, a la derecha del título. En modal van en el footer. En ambos casos el orden es secundario(s) → principal, con el principal siempre al extremo derecho. Botones típicos: Cancelar/Volver, Guardar progreso, Finalizar/Guardar.

Validación de campos obligatorios

Al intentar Finalizar, los campos obligatorios vacíos se marcan con borde rojo y un mensaje de error bajo el campo. (Pruébalo en el demo de vista completa dejando Nombre u Objetivo en blanco.)

Validación por tipo de campo

Cada tipo aplica su propia regla (pruébalo en el Catálogo de tipos de campo): Texto bloquea números en vivo; Numérico solo acepta dígitos; Teléfono admite dígitos y + ( ) -; RUT se autoformatea (puntos y guion) y valida el dígito verificador; Email y Contraseña validan formato y largo al perder el foco. Un texto de ayuda describe la regla y el error aparece solo cuando corresponde.

Mensajes Estable

Modales de estado para comunicar el resultado de una acción o pedir confirmación. Hay cuatro tipos —Éxito, Error, Advertencia e Informativo— cada uno con el ícono y color semántico de los Iconos de Estado y una línea de acento bajo el título. Los botones siguen el estándar de la Suite: secundario → principal, alineados a la derecha.

Tipo = ícono + color semántico

Cada modal usa el ícono de estado correspondiente con su color: Éxito (message-success, verde), Error (message-error, rojo), Advertencia (message-warning, ámbar) e Informativo (information, azul). Una línea de acento bajo el título refuerza el tipo.

Orden de botones: secundario → principal

Las acciones van alineadas a la derecha en el footer, con el botón principal siempre en el extremo derecho. En confirmaciones: Cerrar/Cancelar (secundario) + acción (principal).

Una acción vs. confirmación

Los mensajes de resultado (Éxito, Error, Informativo) suelen llevar un único botón principal (OK, Cerrar, Entendido). La Advertencia se usa para confirmar acciones reversibles/sensibles y lleva dos botones.

Cierre del modal

El modal se cierra con su botón de acción, con la tecla Esc o haciendo clic en el fondo oscurecido. Bloquea el scroll de la página mientras está abierto.

Controller — sap.m.MessageBox

sap.ui.require(["sap/m/MessageBox"], function (MessageBox) {

  // Éxito
  MessageBox.success("Gestión manual registrada.", { title: "Éxito" });

  // Error

    title: "Error al rectificar" });

  MessageBox.information("El proceso fue actualizado por otro usuario.");

    title: "Resolver solicitud",

    emphasizedAction: "Resolver",

  });

});
